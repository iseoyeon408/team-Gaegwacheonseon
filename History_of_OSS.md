## OSS의 역사
-오픈소스 소프트웨어의 역사는 공유 중심의 연구 문화에서 시작해 '자유 소프트웨어 운동 -> Linux 등장 -> 오픈소스 개념 확립 -> 산업 표준'으로 확장되어 왔다.
-현재는 ai, 클라우드, 모바일 os 등 핵심 기술의 기반이 되고 있다.

### OSS 역사 흐름 요약

| 시대 | 핵심 변화 |
|------|----------|
| 1970~1980 | 공유 연구, 해커 윤리 |
| 1983~ | Free Software 운동, 자유 철학 |
| 1991~ | Linux 탄생, 커뮤니티 개발 확립 |
| 1998~ | OSS 용어 정착, 기업 참여 |
| 현대 | AI·클라우드·산업 표준 기술 |

### 1. 1970~1980: 초기의 공유 문화와 해커 윤리
- 컴퓨터는 주로 연구기관, 대학에서만 사용되던 시기
- 개발자들은 프로그램을 서로 공유하며 수정하고 발전시킴
- MIT AI Lab을 중심으로 “기술은 공유될 때 발전한다”는 해커 문화 형성
- 소프트웨어는 당연히 나누는 지식이라는 인식이 일반적

### 2. 1983~: GNU 프로젝트와 Free Software 운동
- 상업화가 진행되며 소프트웨어가 비공개·유료화되는 현상 증가
- 1983년, 리처드 스톨먼이 GNU 프로젝트 선언
- 1985년 FSF(Free Software Foundation) 설립
- “Free”는 무료가 아니라 자유(Freedom)를 의미
- 스톨먼이 주장한 소프트웨어의 4가지 자유
  1. 실행의 자유
  2. 분석 및 수정의 자유
  3. 재배포할 자유
  4. 개선하여 공유할 자유
 
### 3. 1991: 리누스 토르발스와 Linux 탄생
- 대학생 리누스 토르발스가 개인 취미로 운영체제를 개발
- 인터넷에 소스코드 공개 → 전 세계 개발자들이 참여해 개선
- GNU에서 만들어진 도구들과 결합해 완전한 운영체제 구축
- “하비 프로젝트가 글로벌 시스템이 되었다”는 대표적 사례

### 4. 1998: ‘Open Source Software’ 용어 도입
- “Free Software”가 무료 의미로 오해되는 문제 발생
- 기업 이해와 협업을 위해 새로운 용어 필요
- 같은 해, 넷스케이프가 웹 브라우저 소스를 공개하며 논의 촉진
- OSI(Open Source Initiative) 설립, 오픈소스 정의 표준화

### 5. 2000 이후: 글로벌 산업 기반으로 확장
- Apache, Linux, Android 등 대형 프로젝트 성장
- GitHub(2008) 등장 → 협업 능력이 폭발적으로 확대
- AI와 머신러닝: TensorFlow, PyTorch 모두 오픈소스
- 클라우드: Kubernetes가 표준화
